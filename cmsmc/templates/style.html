CMCSS

A reference and implementation guide for curtmerrill.com

Layout

This is a limited-use, focused and opinionated stylesheet. There is one primary column for content, established with the container class.

Through the power of CSS grids, the direct descendants of any .container element are positioned and sized to the center column.

Specific elements may extend beyond the main column, thanks to subgrid. (See Media/figures below.)

Colors

Base colors are defined using Oklch notation and assigned to variables in the stylesheet's :root element.

I try not to use the base color variables directly. I create semantically named variables that reference them. And because they are in Oklch notation, it is relatively easy to modify them.

My hope is that this technique will make it easier to enable dark mode.

Typography

Font sizes are calculated using rem units and clamp() functions generated from Utopia.

Headers

The level 1 heading tag, <h1>, is reserved for page and post titles. Levels 2 and 3 are for subheads. Levels 4-6 are styled the same because they are rarely used.

All heading tags are also available in class-form—.h2, for example—so they can be used for appearance’s sake without semantic meaning.

Page and post titles are wrapped in an <hgroup> tag, to control the spacing of metadata and subtitles.

Media/figures

The figure element should be used with media elements including images, videos and tables.

The <figure> tag has two optional classes, full-width and popout. Using those classes extends the primary element of the figure beyond the edges of the content column.

And the first child of a figure element can have the border class. Using the border class on the primary element adds a border and padding. The demonstration of inline styles earlier in this document is an example of the border class in use.

Tables

Tables should be contained within a figure element. Table headers and footers should be in thead and tfoot elements respectively, and the body should be in the tbody element.

The default styling is a heavy border at the top and bottom, a medium border separating the body from the header and footer, and light borders between rows.

There are examples of tables earlier in the document.

Buttons

Buttons should be treated as block elements. Their vertical size precludes them from fitting well within a line of text.

Archive lists

Lists of previous entries are coded as unordered lists with the archive-list class. The links inside the list items have the plain class, and the dates are span elements with the class, archive-list__date.
